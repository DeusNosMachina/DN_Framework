# Expressions

**Interactive experiences that embody the DN Framework's dimensional architecture.**

Expressions are not explanations of the framework — they are the framework made experiential. Each piece translates DN's structural claims into something you can feel through interaction: cursor movement as attention, click as conscious attestation, entropy as the cost of looking away.

These are built using the [DN Concept-to-Canvas](../skills/SKILL_DN-Concept-to-Canvas.md) skill methodology, grounded in the [DN Kernel](../kernel/DN_Kernel.md), and designed to pass the **embodiment test**: strip all labels, remove all DN vocabulary from the interface, and the visualization should still feel dimensional.

---

## What's Here

Each `.html` file in this directory is a self-contained, dependency-free interactive experience. Open any of them in a browser. No build step, no install, no framework required.

Every expression includes a "How to See" panel accessible from the interface itself, explaining its interactions and what you're experiencing.

---

## Creating New Expressions

Expressions are single-file HTML artifacts. To create one:

1. Start with a concept, question, or experience
2. Use the [DN Concept-to-Canvas skill](../skills/SKILL_DN-Concept-to-Canvas.md) to analyze the concept through the dimensional architecture
3. Map the analysis to interactive parameters
4. Build a working single-file HTML with p5.js or vanilla Canvas
5. Apply the embodiment test: does it feel dimensional without labels?

Contributions welcome. See the [main README](../README.md) for contribution guidelines.
